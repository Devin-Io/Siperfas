Tentang SIPERFAS

Proyek UAS mata kuliah Pemrograman Web (Paralel 2), Universitas Lambung Mangkurat, yang mengangkat studi kasus Website Pengajuan Permintaan untuk perbaikan fasilitas kampus.

Tujuan Website

Menyederhanakan alur laporan kerusakan fasilitas

Di lingkungan kampus, kerusakan fasilitas sering kali dilaporkan secara manual — melalui pesan pribadi, grup kelas, atau langsung kepada petugas. Cara ini kurang terstruktur sehingga laporan dapat tercecer, tidak lengkap, atau sulit dipantau kembali.

SIPERFAS hadir sebagai sebuah simulasi website yang membantu mahasiswa mengisi laporan perbaikan fasilitas secara lebih rapi, mudah, dan terarah. Setiap laporan otomatis mendapat ID, kategori, dan tingkat urgensi sehingga lebih mudah dipantau.

Melihat simulasi daftar pengajuan yang masuk.
Dosen/Tenaga Pengajar
Merasakan manfaat dari fasilitas kampus yang lebih terawat.
Batasan Sistem

Ruang lingkup proyek

1
Tanpa backend
Tidak ada server, API, atau pemrosesan sisi server.
2
Tanpa database
Tidak menggunakan MySQL, MongoDB, atau database persisten lainnya.
3
Data hanya simulasi
Disimpan sementara di localStorage browser.
4
Berjalan langsung di browser
Cukup dibuka dengan file HTML — tanpa proses build atau hosting khusus.
5
Responsif & konsisten
Tampilan menyesuaikan ukuran layar, semua halaman terintegrasi melalui navbar.
Teknologi yang Digunakan

Stack sederhana, fokus pada fundamental

Seluruh website dibangun memakai materi yang sudah diajarkan di mata kuliah Pemrograman Web Paralel 2.

HTML5

Membentuk struktur dan kerangka semua halaman: navbar, hero, form, kartu daftar, dan footer.

CSS3 Custom

Mengatur identitas visual: palet warna teal-amber, typography Bricolage Grotesque, animasi, dan tampilan kartu.

Bootstrap 5

Komponen wajib: Navbar, Grid System, Card, Form, dan Button. Plus modal sukses pasca-submit.

JavaScript

Validasi form, character counter, animasi statistik counter, dan simulasi penyimpanan via localStorage.

jQuery 3.7

Event handling, animasi fade in/out toast notification, dropdown dinamis fakultas → prodi, filter chip, dan smooth scroll.

Responsive Design

Breakpoint Bootstrap (sm/md/lg/xl) memastikan tampilan nyaman di desktop, tablet, maupun handphone.

FAQ

Pertanyaan yang sering ditanyakan

Tidak. SIPERFAS adalah simulasi tanpa backend dan tanpa database. Data hanya tersimpan sementara di localStorage browser pengguna, sehingga akan hilang jika cache browser dibersihkan.

Tinggi dipakai untuk kerusakan yang sangat mengganggu kegiatan kuliah tapi tidak membahayakan. Darurat khusus untuk kondisi yang berpotensi membahayakan keselamatan (korsleting listrik, plafon ambruk, pintu darurat tidak bisa dibuka, dsb).

Setelah berhasil mengirim, kamu akan mendapat nomor ID pengajuan (cth. LPR-2026-007). Buka halaman Daftar Pengajuan lalu cari berdasarkan ID atau nama untuk melihat status terkini.

Belum. SIPERFAS adalah prototipe simulasi untuk keperluan akademik (UAS Pemrograman Web). Agar dapat dipakai dalam skala kampus, masih diperlukan backend untuk menyimpan data permanen, sistem otentikasi, dan integrasi dengan unit pemeliharaan fasilitas.
